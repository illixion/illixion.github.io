What this is

These are the public keys authorized to log into my machines, distributed with end-to-end signature verification. A small updater fetches a signed manifest and checks it against a hardware-backed key before touching authorized_keys.

So it doesn't matter if this website, its CDN, or its host are compromised: a tampered file is rejected, because the signing key never leaves a YubiKey and an offline backup. Don't curl the raw list into authorized_keys — use the updater.

Install the updater

Download → verify the checksum → run install (schedules periodic checks and updates immediately).

# target: macos-arm64 (Apple Silicon) or macos-amd64 (Intel)
URL=https://ssh.illixion.com/bin/ssh-keys-updater-macos-arm64
curl -fLo sku "$URL"
shasum -a 256 sku            # compare to your out-of-band value
chmod +x sku && ./sku install ssh.illixion.com
# target: linux-amd64 or linux-arm64
URL=https://ssh.illixion.com/bin/ssh-keys-updater-linux-amd64
curl -fLo sku "$URL"
sha256sum sku               # compare to your out-of-band value
chmod +x sku && ./sku install ssh.illixion.com

Run from an elevated PowerShell — the admin keys file lives in C:\ProgramData\ssh.

# target: windows-amd64.exe or windows-arm64.exe
$u = "https://ssh.illixion.com/bin/ssh-keys-updater-windows-amd64.exe"
curl.exe -fLo sku.exe $u
certutil -hashfile sku.exe SHA256     # built-in; compare to your value
.\sku.exe install ssh.illixion.com

Installs to /etc/dropbear/authorized_keys; survives sysupgrade.

URL=https://ssh.illixion.com/bin/ssh-keys-updater-openwrt-ramips
uclient-fetch -O /tmp/sku "$URL"      # or: wget -O /tmp/sku "$URL"
sha256sum /tmp/sku                     # compare to your out-of-band value
chmod +x /tmp/sku && /tmp/sku install ssh.illixion.com

The sha256/certutil step above compares against a hash you carry out-of-band. No recorded hash? Verify provenance instead (any platform, needs the gh CLI):

gh attestation verify ssh-keys-updater-<target> --repo illixion/illixion.github.io

Verify the download

The updater carries the trust anchor compiled in, so getting an authentic copy is the one step that matters. The website is untrusted — so don't trust the download, trust the checksum.

Compare the SHA-256 against a value you carry independently — a copy on another device or a mobile note. When you're at a fresh machine with only a keyboard, typing the URL by hand, this is what makes the download safe: the hash, not the channel. Hex is case-insensitive, so uppercase Windows output matches lowercase Unix output.

macOS          shasum -a 256 <file>
Linux / WRT    sha256sum <file>
Windows        certutil -hashfile <file> SHA256   (built in, no install)

Or verify build provenance

Each binary carries a signed SLSA attestation (built by this repo's CI from a known commit). With the gh CLI:

gh attestation verify ssh-keys-updater-macos-arm64 --repo illixion/illixion.github.io

Defense-in-depth — the attestation trusts GitHub, so the out-of-band hash above stays the ultimate check.
